Notes

  1. Had family and friends test the app and conducted an informal usability study, also with family and friends.
    1. Uncovered a few bugs, which I fixed.
    2. It became very clear, early on, that the "nearly" checkbox to the right of the "Rhymes with" input field was more confusing than helpful.
    3. Demographic: Every adult female tester (including one transgender born female) asked what the score in the results means. Not one of the male testers even mentioned it. Every online rhyming dictionary I've seen has some sort of scoring mechanism to indicate how well a result word rhymes with the search word. The same applies for synonyms, etc. in the Datamuse API. There were five female testers, including one teen, and 5 male testers, all adults.
    4. The most-requested feature is to add searching for antonyms.
  2. There is a map of Grading Rubrics to Code in the Functional Description document to help the instructor with grading. The map contains a section for each grading rubric, with subsections as needed, and points to the code at the module, function, method, and event handler level.
