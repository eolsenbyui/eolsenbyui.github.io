Notes

  1. Mixins are an interesting way to work around the lack of multiple inheritance in the language.
  2. As of version 13.2.0, NodeJS supports ECMAScript modules and the import statement in addition to the familiar NodeJS require function. The current stable version of NodeJS is 14.18.0.
  3. The current ECMAScript 2022 language specification includes truly private identifiers. Prefixing an identifier with a hash character ('#') makes it private. When released next year, this will give us private methods and member variables in classes. Yay!
