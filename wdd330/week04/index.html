Notes

  1. Mixins are an interesting way to work around the lack of multiple inheritance in the language.
  2. As of version 13.2.0, NodeJS supports ECMAScript modules and the import statement in addition to the familiar NodeJS require function. The current stable version of NodeJS is 14.18.0.
  3. The current ECMAScript 2022 language specification includes truly private identifiers. Prefixing an identifier with a hash character ('#') makes it private. When released next year, this will give us private methods and member variables in classes. Yay!
  4. Great discussion on iterators in Eloquent JavaScript!
  5. Haverbeke, in Eloquent JavaScript, describes tangled code as a "big ball of mud," which is an apt visual. Throughout my career, I have heard the metaphor "bowl of spaghetti" used to describe disorganized code. Several years ago, a mid-level software engineer created such a monster where I was working at the time. I was a software test manager at that time. (Some people call software testing "quality assurance") but I think that is a misnomer: Testers cannot assure quality though they can help improve it, and besides, quality is everyone's job, not just the testers.) When my test engineers would report a bug, he would fix the bug but create 5 or 6 more in the process of fixing the one bug. After a couple of weeks of this, the development was turned over to a principal software engineer, who re-designed the whole thing from the ground up and had it running in a few days, with an occasional and isolated bug here and there, which was quickly fixed and had no side-effects. The principal engineer's approach was to identify the discrete components first, then design the interfaces, and then implement them in classes.

Comments

  1. Haverbeke claims, in Eloquent JavaScript, Chapter 6, that inheritance creates "more tangle." Twenty-seven years of software development with classes and objects, incluing inheritance and multiple inheritance, tell me that this is not necessarily so. Thoughtfully designed inheritance structures need not be tangled at all but rather, can be clean and clear. I have seen programmers break encapulation and make a mess of polymorphism, just as I have seen programmers abuse inheritance. I might accept the argument that inheritance may be easier to abuse than encapsulation and polymorphism but cannot agree with the author that inheritance inherently creates "more tangle." I do agree with the author, however, that we probably shouldn't actively go looking for opportunities to construct class hierarchies. The need for such will usually be obvious, if not painfully so.
  2. I was disappointed to discover that JavaScript does not support operator overloading. There is a formal proposal to add the feature but it is not included in the ECMSScript 2022 Language Specification. Maybe next time.
  3. Maybe some future version of ECMAScript will offer direct multiple inheritance. Most serious object-oriented languages have that feature. The proposal for ECMAScript 2022 does not include multiple inheritance. Nor does the ECMSScript 2022 Language Specification.
  4. Tic-Tac-Toe seems to be a popular exercise for programming classes and tutorials. I have done several. The instructor's version of the exercise is the most concise that I've seen to date.
  5. I wish I had time to do all of the exercises in Eloquent JavaScript.
