  1. I believe that it's important to become a debugging ninja.
  2. The team activity this week was a case of too many cooks in the kitchen. We got it working, though.
  3. The team activity is a great prelude to the To-Do List challenge.
  4. The Model-View-Controller (MVC) pattern is useful in many cases but Microsoft has promoted it as the user interface model since 1992 — one size must fit all. First it was the Microsoft Foundation Classes in 1992 and then ASP.NET MVC in 2009. Last Thursday at work I was given an ASP.NET MVC application to fix, written by someone who no longer works there (of course). It's a miserable piece of code that might have been much simpler without being shoehorned into the MVC pattern.
  5. Once again, this week at work I've been coding in Python and C#. I can't help but think that JavaScript might benefit from native assertions like C# has. Assertions in C and C++ were just macros but in C# they are part of the System.Diagnostics library, which comes with the language.
  6. TypeScript is a great idea as JavaScript is so weakly typed but TypeScript is so strongly typed that I've found it difficult to use.
  7. I feel that I'm making good progress toward my personal goal of being able to put together a useful Web UI as fast as I can build a Windows desktop UI using WinForms.
