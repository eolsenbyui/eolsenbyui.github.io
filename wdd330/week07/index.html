Notes

  1. functionname.call() passes a this pointer to the first argument passed to .call.
  2. The .apply method works the same way but the arguments after the first are in an array.
  3. Properties can be added to functions. Functions in JavaScript are objects, not merely pointers to code.
  4. Properties are lost if a function redefines itself.
  5. Memoization: cached results. I can see uses for this. I could use this in my current Python project at work, and lo, I find that Python 3.2+ supports it natively.
  6. IIFE ("iffy") functions. Clever but easier to accomplish in ES6 with code blocks as in C/C++ or C#.
  7. XMLHttpRequest has been superseded by the Fetch API. The Fetch API is a "living standard." The Fetch API uses promises. The fetch() method can also be used with async/await. See instructions on how to do so here and here.
  8. The .json() method is the most common of AJAX responses. The .redirect() method is not supported in any browser.
